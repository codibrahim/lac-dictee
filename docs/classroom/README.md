## How to Use This Folder

**For tracking classroom progress:**
1. Read the week's **research file** — this is what you need to investigate
2. Check **deliverables file** — this is the acceptance criteria
3. When you complete a deliverable, update the commit hash in the table above
4. Link to the product/ artifact that corresponds to each deliverable

**For post-graduation portfolio:**
- This folder documents the bootcamp *requirements*
- The `docs/product/` folder shows what you *actually built*
- Together, they prove: "I delivered X on schedule in PR #Y"

---

## Delivery Record Format

Each deliverable gets a record like:

```markdown
## [Deliverable Name]

**Due:** Apr 27, 2026
**Acceptance Criteria:**
- [ ] Criterion 1
- [ ] Criterion 2

**Delivered In:** `abc1234` (PR #5)
**Product Link:** [docs/product/file.md](../product/file.md)
```

This creates an immutable audit trail showing *when* you delivered and *where* in the codebase.
