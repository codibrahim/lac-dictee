# Research & Learning Materials: The Self-Reliant Programmer

**Source:** [Google Classroom](https://classroom.google.com/c/ODQ1MDc1NTAxMjM1/a/ODYwNTUxNTE3MDky/details)
**Due: May 4**

---

## Objective

Master the art of independent troubleshooting and technical documentation.

---

## Instructions

- **Documentation is your Best Friend:** Learn how to read API docs (OpenAI, Slack, Stripe) like a pro.
- **The Triage Mindset:** When you get an error, use the **"15-minute Rule"**: struggle for 15 minutes, document what you tried, then seek help.

---

## Content Areas

### Computational Thinking
- Decompose MVP into small, testable steps
- Recognize patterns, abstract the essentials
- Implement each piece with clear, repeatable logic

### Advanced Troubleshooting
- Using debuggers and AI as a "Rubber Duck" to find logic errors

### Technical Writing
- How to write a README.md that looks professional on GitHub

### Security Best Practices
- Managing `.env` files and environment variables
