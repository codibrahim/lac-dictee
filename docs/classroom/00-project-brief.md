## Mission

For three weeks, step out of the "Employee" role and into the **Founder's Mindset**. You are the CEO, Architect, and Developer of your own AI startup.

Identify a real-world "pain point" and build a functional, AI-powered automation that solves it.

> Your peers are your **Board of Advisors** — use them for feedback, but the Definition of Done rests entirely on your shoulders.

---

## Project Eligibility — The "Green Light" Filter

Before committing to an idea, it must pass all four tests:

| Test | Question |
|------|----------|
| **Business Case** | Does this solve a real problem for a specific user? |
| **AI/LLM Core** | Does the solution require an LLM (text reasoning, summarization, extraction, classification)? |
| **Feasibility** | Can you build an MVP in exactly 4 days of deep work (Week 6)? |
| **Data Availability** | Do you have access to the data (PDFs, CSVs, APIs) needed? |

## 3-Week Roadmap

| Week | Phase | Goal |
|------|-------|------|
| Week 5 (Apr 20–27) | Planning / Design Sprint | Architecture Defense: defending tech stack and blueprint |
| Week 6 (Apr 28–May 4) | Building / The Build | Functional MVP: working prototype with at least one complex logic path |
| Week 7 (May 5–11) | Presenting / The Pitch | Graduation Demo: 5-minute pitch and live demo |

---

## Evaluation Criteria (Week 7)

1. **Problem/Solution Fit** — How well did you address the initial pain point?
2. **Technical Robustness** — Does the AI logic handle edge cases without crashing?
3. **User Experience (UX)** — Is the interface intuitive for a non-technical user?
4. **Security & Ethics** — API keys hidden? PII handled safely?
5. **Performance** — Is the automation fast and reliable?
6. **The Pitch** — Can you tell a compelling story about the impact of your tool?

---

## Solo Definition of Done (Baseline)

- [ ] **Logic:** System uses an LLM to perform a task standard code cannot do easily
- [ ] **Persistence:** System logs interactions or saves data to a database/sheet
- [ ] **Security:** Uses `.env` files for all credentials
- [ ] **Documentation:** Professional `README.md` with setup instructions
- [ ] **Stability:** The "Happy Path" works 100% of the time during a live demo
